# 希望する性能
- 無限ホワイトボードであること
- 無限ホワイトボードを複層レイヤー化できること
	- ボード含めたオブジェクトはワールド座標としてのXYに加え、Z座標を持たせる
	- ボードにはワールドZ座標とボードに所属する範囲のZ座標空域を設定し、ボード上のオブジェクトの重なりはその空域内で行われる
	- 空域を越えた場合は上層レイヤーや下層レイヤーのボードに所属することになる
- ホワイトボード含めた全てオブジェクトとして管理できること。オブジェクトはワールド座標（x,y,z)とスクリーン座標を持つ。
- CanvaライクなUIを持つこと
- オブジェクトはタイプによって分類できること
	- 図形オブジェクト（シェイプ）
		- インラインでテキスト入力ができる
		- 図形オブジェクトは枠線を持つこと
			- 枠線非表示化は可能
			- 枠線は太さをピクセル指定で変更できる
			- 枠線はカラー指定ができる（基本色＋任意）
		- 図形オブジェクトは塗りを持つこと
			- 基本色と任意のカラー設定ができる
		- 図形オブジェクトは透過設定ができる
			- 透過アイコンをクリックすると透過スライダーが表示される
	- ラインオブジェクト（直線、曲線、矢印）
		- ラインオブジェクトは太さと色を変更できる。色はカラーコントロールモジュール（上記の図形オブジェクトも含めて）を使う
		- ラインオブジェクトは端末形状を変更できる
		- ラインオブジェクトは図形オブジェクトと接続できる
	- テキストオブジェクト
	- プログラムコンポーネントオブジェクト
		- HTMLやPWAなどのプログラムをWeaveのコンポーネント（あるいはプラグイン）として表示する枠をWeave上に設け、iframeのように表示させることができる。ただしiframeとの違いは表示枠サイズを調整できることと、表示倍率が自在であること
- オブジェクトはグループ化、グループ化解除ができること
	- グループ化のひとつの姿として「エリア」を設定できる
	- エリアはMiroでのフレームと同等であり、エリアはエリア指定で即時表示することができる
	- エリアの活用例として、エリア同士の移動順を指定することでプレゼンテーションのように見せることができる。エリアはXYだけではなくZが違っても移動できる
- オブジェクトは回転ができること
- オブジェクトは複数選択し、整列させることができる
- ホワイトボードに画像ファイルを挿入することができる
	- 画像ファイルも拡大縮小ができる
- ホワイトボードにドキュメントファイルを挿入することができる
	- テキストファイル、mdファイル（レンダリングされたMD表示可能）,Mermaidファイル（レンダリングして表示可能）など
	- Mermaidはレンダリングできるだけではなく、Weaveのオブジェクトで再現する機能
- プリセットコンポーネント
	- Markdownエディタ
	- テキストエディタ
	- 時計・タイマー
- Gemini APIが利用可能
	- Gemini2.5 Flash など
- IndexedDBを使用する
- GitHub連携
	- GitHub ActionsからCloud Runにデプロイできる
- 複数レイヤーのボードを3D視点（カメラ回転可能、拡大縮小可能）で見ることができる
- オブジェクトは複数レイヤーにインスタンスやコピーを生成することができる
- Gitのようにブランチを別レイヤーに作ることができる
	- 使用例　小説のアイディアツリーを分岐させ、別レイヤーでは別進行を継続させる、など
- ボードのエクスポート機能
- ボードのGoogleドライブへのエクスポート機能

# 技術スタック（より推奨されるものがあれば変更可）

## 🛠 Weave 開発推奨スタック

### 1. フロントエンド（描画・UI）

- **フレームワーク: Next.js (App Router)**
- **3D/描画エンジン: React Three Fiber (R3F)**
- **スタイリング: Tailwind CSS 3**
    - UIの「ガワ」を高速で作るために必須です。Tailwind CSS 4より現状ツール群が安定している3を採用

### 2. 状態管理・データ構造

- **状態管理: Zustand** 
- **データの永続化: IndexedDB (Dexie.js)**
### 3. PWA・ビルドツール

- **ビルド: Vite**
- **PWA化: `serwist` (旧 `@vite-pwa/next`)**

### 4. 開発環境
WSL2
bash
Antigravity
Git及びGitHubでバージョン管理

### 5. ユーザーについて
ユーザーはディレクターとして指揮を取りますが、ITスキルはさほどありません。
Google CloudやGitHubの設定やターミナル操作などは丁寧に支援してください。
