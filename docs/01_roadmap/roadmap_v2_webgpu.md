# Weave ロードマップ V2: WebGPU & 空間OSへの進化

## 概要
Weave を単なる「ホワイトボード」から、WebGPU の圧倒的な計算能力を背景にした「空間型ミドルウェア OS」へと進化させるための新ロードマップ。

## Phase 1: エンジン刷新 (WebGPU Migration)
- [ ] **WebGPURenderer の導入**: `three/webgpu` への移行と WebGPU レンダリングの有効化。
- [ ] **WebGL フォールバックの実装**: WebGPU 非対応環境でも自動的に WebGL 2 で動作する堅牢なレンダリング基盤の構築。
- [ ] **シェーダーコードの TSL 移行**: 既存の素材を Three Shading Language (TSL) に適合させ、WebGPU のポテンシャルを最大限引き出す。

## Phase 2: 高速コンピューティング & 大規模描画 (Compute Power)
- [ ] **GPU Compute によるオブジェクト管理**: 数万個のオブジェクトの位置・物理計算を GPU で直接実行。
- [ ] **パーティクル & 物理演算**: 付箋やカードが空間を舞うような、よりリッチで「生きている」オブジェクトの挙動。
- [ ] **高速キャンバスキャプチャ**: WebGPU のバッファを直接利用した、超高解像度かつ爆速のスクリーンショット機能。

## Phase 3: ミドルウェア・エコシステム (Middleware OS)
- [ ] **PWA / HTML コンポーネントの 3D 統合**: 3D 空間内に SUIREN などの他アプリを「ビルボード」や「壁面」として完全統合。
- [ ] **双方向データ連携**: Weave 上の操作が SUIREN に反映され、SUIREN の結果が Weave の空間を拡張する「Middleware OS」としての機能。
- [ ] **カード・スプレッド・エンジン**: カード配置のレイアウト計算を GPU で一瞬で行う、高度な配置アルゴリズム。

## Phase 4: AI 同期 & 生成的空間 (AI sync)
- [ ] **Gemini 同期レイヤー**: AI がボード全体のコンテキストを把握し、Z軸方向に「考察レイヤー」を自動生成する。
- [ ] **プロンプトによる 3D 生成**: テキストから 3D オブジェクトや、空間全体の雰囲気を生成・変更する機能。

## Phase 5: 至高のビジュアル体験 (Premium Aesthetics)
- [ ] **PBR (Physically Based Rendering)**: 現実世界の光を模した、圧倒的な質感（和紙、ガラス、金属）。
- [ ] **高度なポストエフェクト**: 被写界深度、ブルーム、モーションブラーによる 映画のような SF 的演出。
- [ ] **空間オーディオ**: オブジェクトの位置に基づいた、没入感のあるサウンドデザイン。

## Phase 6: リリース & エクスパンド (Ecosystem)
- [ ] **Weave SDK**: ユーザーが独自のミドルウェアを Weave 上に構築できるための開発キット提供。
- [ ] **モバイル / タブレット最適化 (閲覧)**: 3D 空間をどこでも体験できる PWA の完成。
