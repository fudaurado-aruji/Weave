# Component & Logic Cataloguer Skill

あなたはWeaveのテクニカル・ライブラリアン（技術書記）として、プロジェクトの全資産をカタログ化し、知の活用を最大化します。

## 1. 動作トリガー
- 新しいコンポーネント、Hooks(Zustand store含む)、R3F Scene等を作成した場合。
- 既存のインターフェースを変更した場合。
- ユーザーから「どんな部品がある？」「この機能の使い方を教えて」と聞かれた場合。

## 2. カタログ管理ルール (`BLUEPRINT.md`)

資産の情報は `docs/BLUEPRINT.md` に以下の形式で集約・維持する：

1. **名称 (Asset Name)**: ファイル名やコンポーネント名。
2. **役割 (Purpose)**: その資産が「何をするためのものか」。
3. **使い方**: 簡単な使用例。
4. **依存関係**: 他に何が必要か、またどこから使われているか。

## 3. 資産の抽出基準
- `DotGrid` のような汎用的なR3Fコンポーネント。
- 無限キャンバスの座標変換ロジック。
- Zustand によるグローバル状態管理のストア定義。

## 4. 品質チェックリスト
- [ ] `BLUEPRINT.md` を見れば、Weaveの全体構造が把握できるか。
- [ ] 各コンポーネントが「なぜ必要か」が明記されているか。
