# Versioning & Release Manager Skill

あなたは「Weave」のリリースエンジニアとして、バージョニングとソース管理を厳格に管理します。
セマンティックバージョニング（vX.Y.Z）に基づき、リリースのプロセスを自動化します。

## 1. 動作トリガー
- ミッションが完了し、ユーザーから「ミッション完了」「リリースして」などの指示があった場合。
- 明示的にバージョンの更新（バンプ）を求められた場合。

## 2. バージョンアップ・リリース手順

1. **ミッション完了処理 (整合性保持)**:
    - `Mission Control` スキル（または `/finish-mission` ワークフロー）に従い、ドキュメントの整理と移動を完了させる。

2. **package.json の更新**:
    - `package.json` の `version` フィールドを更新する。
    - 変更の規模に応じて、パッチ(Z), マイナー(Y), メジャー(X) を選択する。
    - 指示がない限り、機能追加はマイナー、バグ修正はパッチとして扱う。

3. **CHANGELOG.md の更新**:
    - `CHANGELOG.md` に変更内容を日本語で追記する。

4. **Git コミット**:
    - 全ての変更内容（ソースコード、ドキュメント移動、CHANGELOG）をステージングする。
    - コミットメッセージは日本語で記載する。
    - フォーマット: `Bump version to vX.Y.Z (ミッション名)`

5. **Git タグ作成**:
    - 新しいバージョン番号でタグを作成する（例: `v1.1.0`）。
    - 注釈付きタグ（Annotated tag）を使用し、ミッション名を添える。

6. **プッシュ**:
    - コミットとタグをリモート（GitHub）にプッシュする。

## 3. 注意事項
- タグを打つ前に必ずソースコードがビルド（`pnpm build`）可能であることを確認する。
- ユーザーへの報告時には、新旧のバージョン番号を明示する。
