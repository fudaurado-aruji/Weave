---
description: 完了したミッションを整理し、バージョンの更新とリリースを行う
---

ミッションの全タスクが完了した際に、ドキュメントの整理とGitでのリリースプロセスを一括で実行します。

1. **ドキュメントの最終確認**
   - ミッションフォルダ内の `task.md` のすべての項目にチェックが入っていることを確認する。

2. **ミッションフォルダの移動**
   - ユーザーから許可を得た後、ミッションフォルダ名を `FINISHED_[元のフォルダ名]` にリネームする。
   - フォルダを `docs/00_finished/` ディレクトリに移動する（ディレクトリがない場合は作成する）。

3. **バージョンの決定**
   - `package.json` の現在のバージョンを確認する。
   - 今回の変更内容に基づき、新しいバージョン番号を決定する。
     - マイナー（Y）: 新機能の追加
     - パッチ（Z）: バグ修正や微調整

4. **ビルド確認**
   // turbo
   - `pnpm build` を実行し、ソースコードにエラーがないことを確認する。

5. **リリース操作**
   // turbo
   - `package.json` の `version` を更新する。
   - すべての変更をステージングし、コミットする。
     - メッセージ: `Bump version to vX.Y.Z (ミッション名)`
   - 新しいバージョン番号でタグを作成する（例: `git tag -a vX.Y.Z -m "ミッション名"`）。
   - リモート（origin main）およびタグをプッシュする。

6. **ユーザーへの報告**
   - 完了した旨と、新しいバージョン番号をユーザーに報告する。
