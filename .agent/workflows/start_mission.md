---
description: 新しいミッションを開始し、所定のルールに従ってドキュメントフォルダとテンプレート一式を作成する
---

1.  ユーザーに**ミッション名**（英語・スネークケース、例: `implement_infinite_canvas`）を尋ねる（まだ指定されていない場合）。
2.  ユーザーに**ミッションの概要**（日本語）を尋ねる（まだ指定されていない場合）。
3.  `run_command` を使用して `docs/[ミッション名]` フォルダを作成する。
    - ※既に存在する場合はユーザーに確認する。
4.  以下の3つのファイルを `docs/[ミッション名]/` 直下に作成する。内容は全て**日本語**とする。

    - **`task.md`**
      ```markdown
      # Mission: [ミッション名]

      [ミッションの概要]

      ## Tasks
      - [ ]  <!-- id: 0 -->
      ```

    - **`implementation_plan.md`**
      ```markdown
      # 実装計画: [ミッション名]

      ## 1. 概要
      [実装の目的と背景]

      ## 2. 実装ステップ
      - [ ] Step 1: ...

      ## 3. ユーザーへの影響
      [変更点やメリット]
      ```

    - **`walkthrough.md`**
      ```markdown
      # 変更内容確認 (Walkthrough): [ミッション名]

      ## 1. 成果物
      - [ ] ...

      ## 2. 変更の詳細
      [コード変更のポイント]

      ## 3. 次のステップ
      ```

5.  作成完了をユーザーに報告する。
